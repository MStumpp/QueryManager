Create QueryManager instance

Allocate an instance of the QueryManager in the following way.

QueryManager *manager = [QueryManager instance];

Configure QueryManager instance

You may set the maximum number of concurrent connections for all queues.

[manager setNumberOfMaxConcurrentConnections:10];
  • numberMaxConcurrent: Maximum number of concurrent connections over all queues. Lowest value is 0. Use tUnlimitedConcurrentConnections for unlimited number of connections.

Create Queue instance

Allocate an instance of Queue in the following way.

Queue *queue = [manager initQueueWithPrio:1 andIdentifier:@"queue 1"]
  • prio: Only one prio of same value per manager. Lowest value for priority is 0 (highest priority).
  • identifier: A unique identifier per queue.

Get Queue instance from QueryManager

Queue *queue = [manager getQueueWithIdentifier:@"queue 1"]

Create Query instance

QueryTicket *ticket = [[ConcreteQuery instanceWithQueue:queue] process:(id)data onCompletion:^(int state, id data, NSError *error) {
   ...
}];

Query implementation

@interface ConcreteQuery : Query
@end

@implementation ConcreteQuery
-(void)load:(id)data
{
    ... do stuff, then call
    [self loadedWithData:data andError:nil];
}

-(void)pause:(id)data
{
    ... do stuff, then call
    [self pausedWithData:data andError:nil];
}

-(void)cancel:(id)data
{
    ... do stuff, then call
    [self cancelledWithData:data andError:nil];
}
@end

Feature Candidates

  • queueMaxSize: Maximum size of a queue, meaning number of Query instances in state initial, loading, pausing or paused (Query instances in state loaded, cancelling or cancelled are not count towards this metric). Use tUnlimitedQueueSize for unlimited number of Query instances. (defaults to: tUnlimitedQueueSize)

    [queue setQueueMaxSize:tUnlimitedQueueSize];

  • numberMaxConcurrent: Maximum number of concurrent connections over all queues. Lowest value is 0. Use tUnlimitedConcurrentConnections for unlimited number of connections. (defaults to: tUnlimitedConcurrentConnections)

    [queue setNumberMaxConcurrent:tUnlimitedConcurrentConnections];

  • processOrder: Order of how Query instances will be processed. Use tfifo for first in first out or tlifo for last in first out. (defaults to: tfifo)

    [queue setProcessOrder:tfifo];

  • finishHigherPrioFirst. (defaults to: false) 

    [queue setFinishHigherPrioFirst:TRUE];

  • pauseWhileLoading: Pauses all Query instances which come after the added one, from the other queues as well as the same queue. (defaults to: false)

    [queue setPauseWhileLoading:TRUE];

  • cancelWhileLoading: Cancels all Query instances which come after the added one, from the other queues as well as the same queue. (defaults to: false)

    [queue setCancelWhileLoading:TRUE];
